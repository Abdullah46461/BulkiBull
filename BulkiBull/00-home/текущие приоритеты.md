# Текущие приоритеты

## Сейчас

- Держать Obsidian vault в папке `BulkiBull/`, а инструкции для Codex в корневом `AGENTS.md`.
- Перед крупными задачами читать [[index|Bulki Bull Knowledge Vault]] и релевантные заметки из `atlas/` и `knowledge/`.
- Сохранять важные решения и баги отдельными заметками с wiki-ссылками.
- Считать исходный код и README главным источником истины: MVP после этапов 1-5 уже реализован.
- Учитывать, что mobile уже выбирает фото из галереи телефона и умеет снимать его камерой, а до backend upload временно хранит сжатый `data:image/...` в `photoUrl`.
- Учитывать, что удаление бычка уже доступно из карточки и требует подтверждения, см. [[удаление бычка в MVP требует подтверждения и удаляет историю веса]].
- Учитывать, что mobile уже поддерживает светлую и темную тему, а переключение идет через inline toggle, см. [[mobile тема переключается inline toggle без отдельного экрана]].
- Помнить, что нижняя mobile-навигация теперь живет в app shell, а не внутри отдельных view, см. [[mobile использует глобальный footer tab bar через App shell]].
- Помнить, что desktop web для `Camera.takePhoto` использует [[desktop web использует PWA Elements для Camera.takePhoto]], а iOS Simulator не дает надежной проверки live preview камеры.
- Учитывать, что учет кормов уже показывает период запаса и дату окончания, а для сохраненных данных period приходит из API, см. [[сохраненный период корма рассчитывается в API и отдается готовыми датами]].
- Помнить, что mobile в последние 10 дней до окончания корма ставит ежедневные локальные уведомления и подсвечивает warning-состояние красным.
- Если mobile UI уже обновился в web, но не обновился в iOS Simulator, использовать сценарий из [[Capacitor copy и sync не обновляют уже установленный iOS build в симуляторе]].
- Если footer mobile пропадает на форме или под ним видно контент при скролле, использовать сценарий из [[локальный footer внутри экрана mobile исчезает на формах и просвечивает при скролле]].
- Для дат feed-периода и уведомлений использовать локальный календарный день, а не UTC ISO-срез, см. [[даты периода корма нельзя строить через UTC toISOString на клиенте]].
- Помнить, что API уже принимает увеличенный photo payload, синхронизированный с shared лимитом `PHOTO_DATA_URL_MAX_LENGTH`.
- Помнить, что MVP уже включает email auth с подтверждением почты до входа, см. [[email auth в MVP требует подтверждения почты до входа и использует API_PUBLIC_URL для ссылок]].
- Для проверки письма на телефоне или другом внешнем устройстве не оставлять `API_PUBLIC_URL` на `localhost`, см. [[localhost в verification link из письма не открывается на телефоне]].
- Помнить, что backend уже покрыт интеграционными тестами для `auth`, `/bulls`, весов, `/feeds` и `currentWeight`, см. [[API integration тесты поднимают Nest приложение на отдельной Prisma schema]].
- Помнить, что README и env-документация теперь уже описывают локальный запуск, demo-flow и SMTP без догадок.
