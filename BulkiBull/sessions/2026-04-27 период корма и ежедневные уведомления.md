# Период корма и ежедневные уведомления

## Контекст

Пользователь попросил доработать учет кормов: вместо числа дней показывать период, подсвечивать красным даты при остатке меньше 10 дней и присылать предупреждение на мобильное устройство. По ходу работы выяснилось, что для устойчивой реализации сохраненный расчет нужно считать не только на front, а вынести в общий слой и отдавать из API.

## Что изменено

- Экран кормов и компактная сводка на списке бычков теперь показывают не `N дн.`, а период запаса и дату окончания.
- Если до окончания корма осталось меньше 10 дней или запас уже закончился, период в mobile подсвечивается предупреждающим красным состоянием.
- В нативном mobile добавлены локальные уведомления через `@capacitor/local-notifications`: в последние 10 дней до окончания корма приложение ставит ежедневное напоминание.
- Для сохраненных записей корма API теперь возвращает `dailyConsumptionKg`, `daysLeft`, `periodStartDate` и `depletionDate`.
- Формула расчета доступности корма вынесена в `packages/shared`, чтобы backend и mobile preview использовали один и тот же чистый helper.
- Mobile продолжает считать локально только превью "После сохранения", а отображение уже сохраненных данных опирается на ответ API.
- После изменений выполнялся полный цикл `build -> cap sync ios -> xcodebuild -> simctl install/launch`, чтобы проверить поведение в iOS Simulator, а не только в web.

## Проверки

В этой сессии проходили:

```bash
npm run typecheck
npm run lint
npm run build
cd apps/mobile
npx cap sync ios
xcodebuild -project ios/App/App.xcodeproj -scheme App -configuration Debug -destination 'id=ACD906DF-93BE-49AB-A079-53043EDC4F5A' -derivedDataPath /tmp/bulki-bull-ios-build build
xcrun simctl install booted /tmp/bulki-bull-ios-build/Build/Products/Debug-iphonesimulator/App.app
xcrun simctl launch booted com.bulkibull.app
```
