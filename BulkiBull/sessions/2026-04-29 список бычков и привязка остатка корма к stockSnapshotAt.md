# Список бычков и привязка остатка корма к stockSnapshotAt

## Контекст

Пользователь попросил слегка подправить экран списка бычков, а затем сообщил о баге в расчете периода корма: пример `01.04-10.04` на следующий день превращался в `02.04-11.04`. По ходу фикса выяснилось, что проблема была не только в отображении даты, а в самой модели `FeedStock`: сохраненный остаток был привязан к техническому `updatedAt`, а не к явному бизнес-снимку.

## Что изменено

- На экране `Бычки` над списком добавлен маленький счетчик общего количества животных.
- Иконка аккаунта в header списка бычков перенесена в крайний правый слот.
- Для feed-расчета в Prisma-модель `FeedStock` добавлено поле `stockSnapshotAt`.
- API теперь считает текущий расчетный остаток, `daysLeft` и `depletionDate` от `stockSnapshotAt`, а не от `updatedAt`.
- В `FeedResponse` добавлено поле `stockSnapshotAt`, чтобы точка отсчета периода была явной и доступной клиенту.
- Shared helper `calculateFeedAvailability` и связанный расчет остатка обновлены под новую точку отсчета.
- API integration тест покрывает сценарий "вчера сохранили остаток, сегодня открыли экран" и проверяет, что дата окончания не дрейфует вперед.
- Локальная база обновлена миграцией, а Prisma Client и shared package пересобраны.
