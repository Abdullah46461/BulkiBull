# Завершение MVP этапов 1-5

## Контекст

Пользователь попросил поэтапно создать MVP мобильного приложения для учета бычков на откорме и строго останавливаться после каждого этапа. После завершения этапа 5 пользователь попросил: `Сохранить сессию`.

## Что готово

- Монорепозиторий на npm workspaces и Turbo.
- Backend API на NestJS.
- PostgreSQL через Docker Compose.
- Prisma schema, migration и seed.
- Shared package `@bulki-bull/shared` с Zod-схемами, DTO/types и util-функциями.
- REST API:
  - `GET /bulls`
  - `GET /bulls?search=...`
  - `GET /bulls/:id`
  - `POST /bulls`
  - `PATCH /bulls/:id`
  - `GET /bulls/:id/weights`
  - `POST /bulls/:id/weights`
- Mobile app на Ionic Vue:
  - список бычков;
  - поиск по бирке;
  - карточка бычка;
  - создание бычка;
  - редактирование бычка;
  - добавление веса;
  - история взвешиваний;
  - отображение фото через `photoUrl`.

## Важные решения

- [[текущий вес вычисляется из последней записи взвешивания]]
- [[фото в MVP хранится как photoUrl]]
- [[shared пакет хранит общие типы и константы]]
