# Localhost в verification link из письма не открывается на телефоне

Если письмо подтверждения содержит ссылку вида `http://localhost:3000/auth/verify-email?...`, она может работать на Mac, но не на телефоне.

## Симптом

- письмо приходит успешно;
- Mail.ru или Safari показывают предупреждение про подозрительный сайт или просто не могут открыть страницу;
- после перехода пользователь видит, что `localhost` недоступен.

## Что это значит

Для телефона `localhost` указывает на сам телефон, а не на Mac, где запущен Nest API. Поэтому ссылка из письма физически ведет не туда.

## Что делать

1. Проверить `API_PUBLIC_URL` в `apps/api/.env`.
2. Для теста на телефоне заменить `localhost` на адрес, достижимый из той же сети:
   - LAN IP Mac вроде `http://192.168.x.x:3000`, либо
   - публичный `https`-домен / tunnel.
3. Перезапустить API.
4. Отправить письмо подтверждения заново, потому что старое письмо уже содержит неправильную ссылку.

## Что помнить

- для local web на одном компьютере `localhost` допустим;
- для реального телефона, планшета или внешнего email-клиента нужен внешний адрес;
- для production `API_PUBLIC_URL` должен быть нормальным публичным `https`-доменом.

См. также [[email auth в MVP требует подтверждения почты до входа и использует API_PUBLIC_URL для ссылок]].
