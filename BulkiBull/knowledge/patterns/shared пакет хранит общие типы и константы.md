# Shared пакет хранит общие типы, контракты и бизнес-утилиты

`packages/shared` публикуется как `@bulki-bull/shared` и используется backend и mobile как единый источник контрактов и чистых переиспользуемых бизнес-утилит.

Сейчас в `packages/shared/src/index.ts` есть:

- response-типы API;
- Zod-схемы для создания и редактирования бычка;
- Zod-схема для добавления веса;
- enum-like values для пола бычка;
- feed-контракты для остатков корма;
- утилиты расчета и форматирования возраста;
- чистые функции расчета доступности корма, которые можно вызывать и из API, и из mobile preview.

## Правило развития

Новые DTO, response-типы и чистые бизнес-формулы сначала добавлять в shared, затем подключать в `apps/api` и `apps/mobile`. Это помогает держать API и mobile в одном контракте и не дублировать вычисления в разных слоях.

Если контракт содержит runtime validation, использовать Zod-схему из shared, а не дублировать проверку вручную.

Если одна и та же формула нужна и backend, и клиенту, в shared должен жить чистый helper без UI-побочных эффектов. Для уже сохраненных сущностей клиент по возможности должен отображать готовый результат API, а не пересчитывать его заново.
