# Сохраненный период корма рассчитывается в API и отдается готовыми датами

Для уже сохраненных остатков корма источником истины должен быть backend. API возвращает не только исходные поля остатка и расхода, но и готовые вычисленные значения `dailyConsumptionKg`, `daysLeft`, `periodStartDate` и `depletionDate`.

## Почему так

- mobile и другие будущие клиенты не должны дублировать бизнес-логику сохраненных feed-расчетов;
- один вычислительный слой уменьшает риск расхождения дат и предупреждений между web, mobile и нативными уведомлениями;
- серверный контракт проще тестировать и развивать, чем несколько независимых front-реализаций;
- это позволяет хранить preview-логику на клиенте только для несохраненного состояния формы, не смешивая ее с каноничными данными.

## Как это устроено

- чистая формула расчета вынесена в `packages/shared`;
- `apps/api` использует shared helper и отдает готовый period в `FeedResponse`;
- `apps/mobile` читает period из API для сохраненных данных;
- mobile вычисляет локально только предпросмотр "После сохранения", чтобы пользователь видел результат до `PUT /feeds/:type`.
