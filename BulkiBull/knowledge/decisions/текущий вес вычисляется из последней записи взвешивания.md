# Текущий вес вычисляется из последней записи взвешивания

В MVP `currentWeight` не хранится отдельным полем в базе. API вычисляет его из последней записи `WeightRecord`: сначала по `date`, затем по `createdAt`.

Если у бычка еще нет записей веса, `currentWeight` равен `initialWeight`.

## Почему так

Для MVP это проще и надежнее: не нужна синхронизация отдельного поля, нет риска рассинхронизации между `Bull.currentWeight` и историей веса.

## Где смотреть

- `apps/api/src/bulls/bulls.service.ts`
- `apps/api/prisma/schema.prisma`
- `packages/shared/src/index.ts`

## Последствия

Если позже появятся отчеты или массовые выборки по большому поголовью, можно оптимизировать запросы или добавить материализованное поле. До этого история веса остается источником истины.
